# Change: プロジェクトファイル整理とハードコード修正

## Why

プロジェクトのルートディレクトリにテストファイルが散在し、バックアップファイルが残存しています。

**より重大な問題として、コード内に具体的なモデル名（「gpt-oss:20b」「llama3」「llama3.2」など）がハードコーディングされており、ユーザーが設定で選択したモデルを無視して固定値を強制しています。これはバグです。**

コードは設定から動的に取得したモデル名のみを使用すべきであり、デフォルト値として具体的なモデル名を持つべきではありません。

## What Changes

- ルートディレクトリのテストファイルを`tests/`に移動
- バックアップファイル(`.backup`)を削除
- `migrate_db.py`を`scripts/`ディレクトリに移動
- **すべての具体的なモデル名をコードから削除**
- **設定値から動的に取得するロジックに統一**
- **デフォルト値は`None`または設定から取得**
- **docstringから具体的なモデル名の例を削除**
- **テストデータから具体的なモデル名を削除**

## Impact

- Affected specs: なし(コードクリーンアップのみ)
- Affected code:
  - ルートディレクトリ: テストファイル6件、`migrate_db.py`、バックアップファイル2件
  - `src/researcher/pages/shared_utils.py`: 初期化ロジック、セッション読み込みロジック
  - `src/researcher/cli.py`: デフォルト値設定
  - `src/researcher/config.py`: docstring
  - `src/researcher/utils/page_utils.py`: 同様のロジック
  - `tests/`: 複数のテストファイル全体
