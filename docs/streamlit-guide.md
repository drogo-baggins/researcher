## セットアップと起動

### 前提条件

- Python 3.9+
- Ollama (LLM モデルを実行するため)
- SearXNG (検索機能用)

詳細は [メインの README](../README.md) を参照してください。

### インストール

```bash
# researcher のインストール
pip install researcher

# または開発モード
pip install -e .
```

### WebUI の起動

```bash
# 最も簡単な方法
researcher-webui

# または手動で起動
streamlit run src/researcher/webui.py

# ポート指定の場合
streamlit run src/researcher/webui.py -- --server.port 8502
```

WebUI は自動的に `http://localhost:8501` でブラウザを開きます。

### シャットダウン

ターミナルで `Ctrl+C` を押すか、ブラウザのタブを閉じます。

### サイドバー

#### セッション管理

**表示内容:**
- セッション一覧（更新日時でソート、最新順）
- セッション検索ボックス
- 新規セッション作成ボタン

**セッション操作:**
- **既存セッション選択**: ドロップダウンで過去のセッションを選択
- **セッション検索**: 名前または会話内容で検索
- **セッション削除**: 選択中のセッション横の🗑️ボタン
- **セッション名変更**: セッションを選択後、テキスト入力で新名前を入力

**例:**
```
📁 セッション
▼ セッション選択: [現在: 新規セッション ▼]
🔍 セッション検索: [質問について ___________]
  📄 質問について (2024-01-20)
  📄 プログラミング (2024-01-19)
  📄 政治ニュース (2024-01-18)
🗑️ 削除

📝 セッション名:
[新規セッション________________]

✏️ 名前を変更
```

#### モデル・言語設定

**モデル選択:**
- Ollama で利用可能なモデルを自動取得してドロップダウンに表示
- モデルを切り替えると、その時点から新しいモデルで回答を生成
- 過去のメッセージは再生成されません（セッション切り替え時のみ反映）

**言語選択:**
- 🇯🇵 日本語 (ja)
- 🇬🇧 English (en)
- セッションごとに保存されます

**接続ステータス:**
- 🟢 すべて接続: Ollama + SearXNG が動作中
- 🟡 部分接続: どちらかが未接続
- 🔴 未接続: 両方が起動していない

#### その他

**「設定を初期化」ボタン:**
- 全セッションの履歴をクリア
- モデル選択をリセット
- 言語設定を初期化

### チャットエリア

**メッセージ表示:**
- **アシスタント**: 青背景、回答テキスト、検索に基づく内容
- **ユーザー**: 右寄せ、入力したクエリを表示
- **メッセージストリーミング**: 回答はリアルタイムで表示される

**引用表示:**
- 回答の下に「📎 引用」セクションが表示されます
- クリックで展開し、検索結果のタイトルと URL を表示

**メッセージ入力:**
- テキストボックスに質問を入力
- Enter キーで送信、または右側のボタンをクリック
- 送信後、自動的に SearXNG で検索が実行されます

**例:**
```
🤖 アシスタント:
ユーザーのクエリに基づいて検索結果から情報を収集しています...

You: 最新のニュースは何ですか？

📎 引用 (5)
[展開]

[テキストボックス]
> _
[送信]
```

## セッション管理

### セッションの特性

- **自動保存**: メッセージ送信後、セッションが自動的に SQLite データベースに保存されます
- **永続性**: WebUI を終了してもセッションは保存されます
- **メタデータ保存**:
  - セッション名
  - 使用していたモデル
  - 使用言語
  - チャット履歴（JSON 形式）
  - 作成日時・更新日時

### セッションデータの場所

セッションデータは以下の場所に SQLite データベースとして保存されます:

```
~/.researcher/sessions.db
```

**手動でセッションをバックアップ:**

```bash
cp ~/.researcher/sessions.db ~/.researcher/sessions.db.backup
```

**セッションの復元:**

```bash
cp ~/.researcher/sessions.db.backup ~/.researcher/sessions.db
```

### セッション検索

**検索機能の詳細:**
- セッション名と会話内容の両方を全文検索
- 検索結果は最大 5 件表示
- 結果をクリックすると、そのセッションがロードされます

**例:**
```
検索結果:
📄 プログラミングの質問 (2024-01-20)
📄 Python について (2024-01-19)
📄 プログラム実行方法 (2024-01-18)
```

## 設定オプション

### Streamlit 設定ファイル

`~/.streamlit/config.toml` を編集して、WebUI の動作をカスタマイズできます:

```toml
[client]
showErrorDetails = false

[logger]
level = "info"

[server]
port = 8501
headless = true
runOnSave = false

[theme]
primaryColor = "#1f77e4"
backgroundColor = "#ffffff"
secondaryBackgroundColor = "#f0f2f6"
textColor = "#262730"
```

### ログレベルの変更

researcher モジュールのログレベルを変更する場合:

```python
# コード内で（または起動前に環境変数を設定）
import logging
logging.getLogger("researcher").setLevel(logging.DEBUG)
```

または環境変数で:

```bash
export RESEARCHER_LOG_LEVEL=DEBUG
researcher-webui
```

## トラブルシューティング

### WebUI が起動しない

**症状:** `streamlit run` 実行後、ブラウザが開かない

**解決方法:**

1. ポート 8501 が使用中かどうか確認:
   ```bash
   lsof -i :8501
   ```

2. 別のポートを指定して起動:
   ```bash
   streamlit run src/researcher/webui.py -- --server.port 8502
   ```

3. ポートを強制的に解放（macOS）:
   ```bash
   kill -9 $(lsof -t -i :8501)
   ```

### Ollama 接続エラー

**症状:** 「🔴 未接続」と表示される、またはモデル選択が空

**原因:** Ollama が起動していない

**解決方法:**

```bash
# Ollama の起動確認
ollama serve

# 別のターミナルでモデルが利用可能かチェック
ollama list
```

**症状:** 「モデルが見つかりません」

**解決方法:**

1. Ollama にモデルをプル:
   ```bash
   ollama pull llama2
   ollama pull mistral
   ```

2. WebUI をリロード (F5 キー)

### SearXNG 接続エラー

**症状:** 検索が失敗する、引用が表示されない

**原因:** SearXNG が起動していない

**解決方法:**

```bash
# SearXNG の起動確認
docker run -d -p 8888:8888 searxng/searxng
```

### セッションが保存されない

**症状:** WebUI を閉じると、作成したセッションが消えている

**原因:** セッションディレクトリの権限不足

**解決方法:**

```bash
# ディレクトリ権限の確認
ls -la ~/.researcher/

# 必要に応じて権限を修正
chmod 755 ~/.researcher
chmod 644 ~/.researcher/sessions.db
```

### メモリ不足エラー

**症状:** 大規模なセッションでメモリエラーが発生

**解決方法:**

1. 古いセッションを削除
2. Ollama のモデルサイズを削減
3. システムメモリを確認:
   ```bash
   free -h
   ```

### ブラウザがエラーを表示

**症状:** `StreamlitAPIException` や `DeltaGenerator` エラー

**解決方法:**

1. ブラウザをハード再読込（Shift + F5）
2. ブラウザキャッシュをクリア
3. WebUI を再起動:
   ```bash
   # ターミナルで Ctrl+C を押す
   researcher-webui
   ```

## セキュリティに関する注意

### 重要な制限事項

⚠️ **Streamlit WebUI はローカル個人使用を想定して設計されています。**

以下の点にご注意ください:

1. **ローカル環境のみ:**
   - デフォルトでは `localhost:8501` にのみバインドされます
   - インターネット経由でアクセスすることは想定されていません

2. **認証機構がありません:**
   - WebUI にはユーザー認証機能がありません
   - ローカルネットワーク上で公開すると、誰でもアクセス可能です

3. **MCP ツール経由のファイルアクセス:**
   - MCP (Model Context Protocol) ツールを通じて、LLM がファイルシステムにアクセスできます
   - 信頼できないプロンプトを実行しないでください

4. **セッションデータの暗号化:**
   - セッションデータ（会話履歴）は平文で SQLite に保存されます
   - `~/.researcher/sessions.db` は暗号化されていません

### 推奨される運用方法

- **単一ユーザー、単一マシンでの使用を前提**
- **機密情報をプロンプトに入力しない**
- **ローカルファイルアクセスが必要な場合は、慎重に検討**
- **定期的なバックアップ**: `~/.researcher/sessions.db` をバックアップ

### ネットワークにおける使用に関する注意

共有ネットワーク環境で WebUI を使用する場合:

```bash
# 例: VPN 接続されたマシンでのみアクセス可能にする場合
# これでも認証はないため、VPN 内の全員がアクセス可能です
streamlit run src/researcher/webui.py -- --server.port 8501
```

**強く推奨**: VPN や SSH トンネルなどの追加のセキュリティレイヤーを使用してください。

### セッションデータの管理

セッションデータが含む内容:

- ユーザーのクエリ（検索内容）
- LLM の回答
- 使用したモデルと言語設定
- タイムスタンプ

**削除方法:**

```bash
# 特定のセッションの削除: WebUI の UI から行えます
# 全セッションの削除: 「設定を初期化」ボタンまたは以下のコマンド
rm ~/.researcher/sessions.db
```

## よくある質問（FAQ）

### Q: 複数のマシンからセッションを共有できますか？

A: 現在、セッションデータはローカルマシンの `~/.researcher/sessions.db` に保存されます。複数マシン間での共有はサポートしていません。

### Q: セッションをエクスポートできますか？

A: 手動でデータベースをコピーするか、実装予定の CSV エクスポート機能をお待ちください。

### Q: ダークモードはありますか？

A: `~/.streamlit/config.toml` で `theme.base` を `dark` に設定できます。

### Q: インターネット経由でアクセスできますか？

A: セキュリティ上の理由から、デフォルトではローカルのみです。インターネット経由でのアクセスには、リバースプロキシと認証機構の設置が必要です。
