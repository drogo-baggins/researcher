## セットアップと起動

### 前提条件

- Python 3.11+
- Ollama (LLM モデルを実行するため)
- SearXNG (検索機能用、オプション)

詳細は [メインの README](../README.md) を参照してください。

### インストール

```bash
# researcher のインストール
pip install researcher

# または開発モード
pip install -e .
```

### WebUI の起動

**推奨方法（マルチページ対応）:**

```bash
# コマンドラインから起動
researcher-webui

# または手動で Home.py を起動
streamlit run src/researcher/Home.py

# ポート指定の場合
streamlit run src/researcher/Home.py -- --server.port 8502
```

**レガシー方法（非推奨）:**

```bash
# 旧 webui.py（互換性シムとして残存）
streamlit run src/researcher/webui.py
# → Home.py にリダイレクトされます
```

> **注意**: `webui.py` は互換性のために残されていますが、将来のバージョンで削除される予定です。
> 新しい `Home.py` エントリーポイントを使用してください。

WebUI は自動的に `http://localhost:8501` でブラウザを開きます。

### シャットダウン

ターミナルで `Ctrl+C` を押すか、ブラウザのタブを閉じます。

## マルチページアーキテクチャ

### ページ構成

Researcher WebUI は3つのページで構成されています:

1. **🏠 Home** (`Home.py`)
   - エントリーポイント
   - セッション初期化
   - ウェルカム画面とナビゲーション

2. **💬 Chat** (`pages/1_💬_Chat.py`)
   - チャットインターフェース
   - LLM との対話
   - ミニマルサイドバー（モデル/言語/自動検索設定のみ）
   - 評価スコア表示

3. **📚 History** (`pages/2_📚_History.py`)
   - セッション管理
   - グループ管理
   - 検索・フィルタリング
   - カレンダー可視化

### ナビゲーション

#### ページ間の移動

- **サイドバー上部**: Streamlit のページセレクター（🏠 Home / 💬 Chat / 📚 History）
- **直接リンク**: Chat ページのサイドバーに「📚 履歴」リンク
- **セッション閲覧**: History ページで selectbox からセッション選択 → 同ページ内で詳細表示（読み取り専用）

> **注意**: History ページは読み取り専用です。セッションを選択しても Chat ページへの自動遷移は行われません。
> チャットを続けるには、サイドバーから手動で Chat ページに移動してください。

#### Deep Link 対応

すべてのページは直接アクセス可能です:

```
http://localhost:8501/               → Home
http://localhost:8501/Chat           → Chat (自動初期化)
http://localhost:8501/History        → History (自動初期化)
```

**ブックマーク対応**: Chat や History ページを直接ブックマークしても、初期化が自動的に実行されます。

## セッション管理

### セッションの特性

- **自動保存**: メッセージ送信後、自動的にSQLiteに保存
- **永続性**: WebUI 終了後も保存
- **メタデータ**:
  - セッション名（自動生成または手動設定）
  - 所属グループID
  - 使用モデル
  - 使用言語
  - 会話履歴（JSON）
  - 検索結果
  - 最終評価スコア
  - タグ（カンマ区切り）
  - 作成日時・更新日時

### セッションデータの場所

```
~/.researcher/sessions.db
```

**手動バックアップ:**

```bash
cp ~/.researcher/sessions.db ~/.researcher/sessions.db.backup
```

**復元:**

```bash
cp ~/.researcher/sessions.db.backup ~/.researcher/sessions.db
```

### セッションのライフサイクル

1. **作成**: Chat ページで最初のメッセージ送信時
2. **更新**: メッセージ追加ごとに自動保存
3. **閲覧**: History ページの selectbox で選択し、詳細を表示（読み取り専用）
4. **編集**: History ページから Chat ページに移動してセッションを継続
5. **削除**: History ページの削除ボタン（実装予定）

> **重要**: History ページでセッションを選択しても Chat ページへの自動遷移は発生しません。
> チャットを続けるには、サイドバーから手動で Chat ページに移動してください。

### グループ管理

**グループの役割:**
- セッションを論理的に分類
- デフォルトグループ（ID: 1）は削除不可
- グループごとにセッション一覧をフィルタ

**グループ操作:**
- **作成**: History サイドバー「新規グループ作成」
- **切り替え**: グループ一覧からボタンクリック
- **名前変更**: 「グループ名変更」セクション
- **削除**: 「グループ削除」セクション（セッションはデフォルトに移動）

## 設定オプション

### Streamlit 設定ファイル

`~/.streamlit/config.toml`:

```toml
[client]
showErrorDetails = false

[logger]
level = "info"

[server]
port = 8501
headless = true
runOnSave = false
address = "127.0.0.1"  # localhost のみ

[theme]
primaryColor = "#1f77e4"
backgroundColor = "#ffffff"
secondaryBackgroundColor = "#f0f2f6"
textColor = "#262730"
```

### ログレベルの変更

```bash
export RESEARCHER_LOG_LEVEL=DEBUG
researcher-webui
```

または Python コード内:

```python
import logging
logging.getLogger("researcher").setLevel(logging.DEBUG)
```

## トラブルシューティング

### WebUI が起動しない

**症状**: `streamlit run` 実行後、ブラウザが開かない

**解決方法:**

1. ポート 8501 が使用中か確認:
   ```bash
   lsof -i :8501
   ```

2. 別のポートを指定:
   ```bash
   streamlit run src/researcher/Home.py -- --server.port 8502
   ```

3. ポートを解放（macOS）:
   ```bash
   kill -9 $(lsof -t -i :8501)
   ```

### Ollama 接続エラー

**症状**: 「🔴 未接続」と表示される

**解決方法:**

```bash
# Ollama の起動確認
ollama serve

# モデル一覧確認
ollama list

# モデルのプル
ollama pull llama2
```

### SearXNG 接続エラー

**症状**: 検索が失敗する

**解決方法:**

```bash
# Docker で SearXNG を起動
docker run -d -p 8888:8888 searxng/searxng

# または設定を確認
curl http://localhost:8888
```

### セッションが保存されない

**症状**: WebUI を閉じるとセッションが消える

**解決方法:**

```bash
# ディレクトリ権限の確認
ls -la ~/.researcher/

# 権限修正
chmod 755 ~/.researcher
chmod 644 ~/.researcher/sessions.db
```

### ページ間遷移が失敗する

**症状**: Chat → History 遷移時にエラー

**解決方法:**

1. ブラウザをハード再読込（Shift + F5）
2. ブラウザキャッシュをクリア
3. WebUI を再起動:
   ```bash
   # Ctrl+C で停止
   researcher-webui
   ```

### Deep Link でセッション初期化エラー

**症状**: Chat/History ページを直接開くとエラー

**解決方法:**

- **v1.2+**: 自動初期化が実装されているため、エラーは発生しません
- **旧バージョン**: Home ページから起動してください

## セキュリティに関する注意

### 重要な制限事項

⚠️ **Streamlit WebUI はローカル個人使用を想定**

1. **ローカル環境のみ**:
   - デフォルトでは `localhost:8501` にバインド
   - インターネット経由アクセスは非推奨

2. **認証機構なし**:
   - ユーザー認証機能なし
   - ローカルネットワーク公開すると誰でもアクセス可能

3. **ファイルアクセス**:
   - MCP ツール経由でファイルシステムにアクセス
   - 信頼できないプロンプトを実行しない

4. **セッションデータ**:
   - 平文で SQLite に保存
   - `~/.researcher/sessions.db` は暗号化なし

### 推奨される運用方法

- **単一ユーザー、単一マシン**
- **機密情報をプロンプトに含めない**
- **定期的バックアップ**: `~/.researcher/sessions.db`

### ネットワーク経由アクセス

**SSH トンネル経由（推奨）:**

```bash
ssh -N -L 8501:localhost:8501 user@remote-machine
# ローカルブラウザで http://localhost:8501 にアクセス
```

**リバースプロキシ + 認証（Nginx）:**

```nginx
server {
    listen 443 ssl;
    server_name researcher.example.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    auth_basic "researcher";
    auth_basic_user_file /etc/nginx/.htpasswd;
    
    location / {
        proxy_pass http://127.0.0.1:8501;
    }
}
```

### セッションデータの管理

**含まれる内容:**
- ユーザークエリ
- LLM の回答
- モデル・言語設定
- タイムスタンプ

**削除方法:**

```bash
# 特定セッション: History ページの UI から
# 全セッション削除:
rm ~/.researcher/sessions.db
```

## よくある質問（FAQ）

### Q: 旧 webui.py との違いは？

A: マルチページ設計により、Chat と History が分離されました。
- **webui.py**: 全機能が1ページ（非推奨、互換性シムのみ）
- **Home.py + Chat.py + History.py**: 機能別に分離、パフォーマンス向上

### Q: 複数マシンでセッション共有可能？

A: 現在、ローカルマシンの `~/.researcher/sessions.db` に保存されます。複数マシン間での共有は未サポート。

### Q: セッションエクスポート機能は？

A: 手動でデータベースをコピーするか、CSV エクスポート機能の実装をお待ちください。

### Q: ダークモードは？

A: `~/.streamlit/config.toml` で `theme.base = "dark"` に設定できます。

### Q: Chat ページを直接ブックマークできる？

A: はい。v1.2+ では自動初期化により、すべてのページが直接アクセス可能です。
